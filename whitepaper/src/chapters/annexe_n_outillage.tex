\section{Stack de Développement (Cyborg-Ready)}

\subsection{Environnement Local}
\begin{itemize}
    \item \textbf{OS :} Linux (Ubuntu/Pop!_OS) ou MacOS. Windows via WSL2 uniquement.
    \item \textbf{IDE :} VS Code (Profile RBK partagé : Extensions Rust, Solidity, GitLens, Copilot).
    \item \textbf{Terminal :} Alacritty + Tmux + Starship (pour la vélocité).
\end{itemize}

\subsection{Chain Stack}
\begin{itemize}
    \item \textbf{Solana :} Rust 1.75+, Anchor 0.29+, Solana CLI 1.18+.
    \item \textbf{EVM :} Foundry (Forge, Cast, Anvil). Pas de Hardhat.
    \item \textbf{Indexing :} The Graph (EVM), Helius/Shyft (Solana).
\end{itemize}

\section{Infrastructure CI/CD (Github Actions)}
Tout repo étudiant doit inclure un workflow `.github/workflows/ci.yml` standardisé :
\begin{itemize}
    \item \textbf{Lint :} `cargo clippy` / `solhint`.
    \item \textbf{Test :} `cargo test` / `forge test`.
    \item \textbf{Audit :} `cargo audit` (dépendances).
    \item \textbf{Format :} `cargo fmt --check`.
\end{itemize}
